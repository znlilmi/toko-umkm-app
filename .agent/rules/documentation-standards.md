# Documentation Standards

## File Naming
- Semua dokumentasi disimpan di folder `docs/`
- UML diagrams di `docs/uml/`
- Format file: `[nomor]-[nama-topik].md` atau `[nama].puml`

## Code Comments
- Setiap file PHP harus memiliki header komentar: tujuan file
- Setiap method yang kompleks harus ada komentar penjelasan
- Gunakan Bahasa Indonesia untuk komentar (agar mudah dibaca)

## Commit Style
- Gunakan Conventional Commits: `feat:`, `fix:`, `docs:`, `refactor:`
- Contoh: `feat: tambah fitur checkout dengan validasi stok`
